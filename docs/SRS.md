## 1. 서론

### 1.1 목적

본 문서는 초등학생용 기하학 AR 학습 웹앱의 소프트웨어 요구사항을 정의합니다. 개발자, 테스터, 프로젝트 관리자가 시스템의 기능과 제약사항을 이해하고 구현할 수 있도록 상세한 명세를 제공합니다.

### 1.2 범위

**제품명**: 도형 왕국 (Geometry Kingdom)

**주요 기능**:
- AR 기반 3D 도형 시각화
- 7개 레벨의 단계별 학습
- 퀴즈 및 평가 시스템
- 게이미피케이션 요소
- 주령구 문화 콘텐츠

**제외 사항**:
- 서버 기반 사용자 인증
- 실시간 멀티플레이어
- 유료 결제 시스템
- 소셜 미디어 통합

### 1.3 정의 및 약어

| 용어 | 정의 |
|------|------|
| AR | Augmented Reality (증강현실) |
| A-Frame | WebVR/AR 프레임워크 |
| AR.js | 웹 기반 AR 라이브러리 |
| LocalStorage | 브라우저 로컬 저장소 |
| 마커 | AR 인식용 패턴 이미지 |
| 주령구 | 통일신라시대 14면체 주사위 |
| MVP | Minimum Viable Product |
| UI | User Interface |
| UX | User Experience |

### 1.4 참조 문서

- PRD (Product Requirements Document)
- TRD (Technical Requirements Document)
- 2015 개정 초등 수학 교육과정
- A-Frame 공식 문서 v1.4.0
- AR.js 공식 문서

## 2. 전체 설명

### 2.1 제품 관점

본 시스템은 웹 기반 독립형 애플리케이션으로, 외부 서버 없이 클라이언트 측에서 모든 기능을 수행합니다.

```
┌─────────────────────────────────────┐
│         사용자 브라우저              │
│  ┌───────────────────────────────┐  │
│  │     프론트엔드 애플리케이션    │  │
│  │  ┌─────────┐  ┌─────────┐    │  │
│  │  │ HTML/CSS│  │JavaScript│    │  │
│  │  └─────────┘  └─────────┘    │  │
│  │  ┌─────────┐  ┌─────────┐    │  │
│  │  │ A-Frame │  │  AR.js  │    │  │
│  │  └─────────┘  └─────────┘    │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │      LocalStorage (데이터)    │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
         ↓ (CDN)
┌─────────────────────────────────────┐
│      외부 리소스 (CDN)               │
│  - A-Frame 라이브러리                │
│  - AR.js 라이브러리                  │
│  - Google Fonts                     │
└─────────────────────────────────────┘
```

### 2.2 제품 기능

#### 주요 기능 그룹

1. **AR 학습 모듈**
   - 마커 기반 AR
   - 마커리스 AR
   - 도형 시각화

2. **교육 콘텐츠 모듈**
   - 레벨 시스템

4. **특별 콘텐츠 모듈**
   - 주령구 시스템

   - 역사 교육

## 3. 시스템 아키텍처

### 3.1 아키텍처 개요

**아키텍처 패턴**: MVC (Model-View-Controller) 변형

```
┌─────────────────────────────────────────────────────┐
│                    Presentation Layer                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │  HTML    │  │   CSS    │  │  A-Frame │          │
│  │  Pages   │  │  Styles  │  │   Scene  │          │
│  └──────────┘  └──────────┘  └──────────┘          │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   Application Layer                  │
│  ┌──────────────┐  ┌──────────────┐                │
│  │ Controllers  │  │   Services   │                │
│  │ - AR         │  │ - Quiz       │                │
│  │ - Level      │  │ - Progress   │                │
│  │ - Game       │  │ - Sound      │                │
│  └──────────────┘  └──────────────┘                │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                     Data Layer                       │
│  ┌──────────────┐  ┌──────────────┐                │
│  │    Models    │  │    Storage   │                │
│  │ - Shape      │  │ - LocalStorage│               │
│  │ - Level      │  │ - JSON Files │                │
│  │ - User Data  │  │              │                │
│  └──────────────┘  └──────────────┘                │
└─────────────────────────────────────────────────────┘
```

### 3.2 모듈 구조

#### 3.2.1 Core Modules

```
js/
├── app.js                    # 애플리케이션 초기화 및 라우팅
├── utils.js                  # 공통 유틸리티 함수
└── progress-tracker.js       # 진행도 추적 및 저장
```

```
js/
```

#### 3.2.3 Game Modules

```
js/
├── level-manager.js          # 레벨 시스템 관리

├── shop-system.js            # 상점 시스템
└── collection-manager.js     # 도감 관리
```

#### 3.2.4 Special Modules

```
js/
├── juryeonggu.js             # 주령구 시스템
├── shape-builder.js          # 도형 창작 시스템

```

#### 3.2.5 UI Modules

```
js/
├── ui-controller.js          # UI 인터랙션

```

### 3.3 데이터 흐름

```
사용자 입력 (터치/클릭)
    ↓
UI Controller
    ↓
Application Controller
    ↓
Service Layer
    ↓
Data Model
    ↓
LocalStorage / JSON

사용자 피드백 (시각/청각)
```

## 4. 기능 요구사항

### 4.1 AR 시스템

#### FR-AR-001: 마커 기반 AR 초기화
**우선순위**: 높음  
**설명**: 사용자가 AR 페이지에 접속하면 카메라를 활성화하고 마커 인식을 시작한다.

**입력**:
- 사용자의 페이지 접속
- 카메라 권한 허용

**처리**:
1. A-Frame 씬 초기화
2. AR.js 마커 감지 시작
3. 카메라 스트림 활성화
4. 마커 등록 (Hiro, Kanji)

**출력**:
- 카메라 뷰 표시
- 마커 감지 상태 UI 표시

**예외 처리**:
- 카메라 권한 거부 시 에러 메시지 표시
- 브라우저 미지원 시 안내 메시지

#### FR-AR-002: 도형 표시
**우선순위**: 높음  
**설명**: 마커가 감지되면 해당 레벨의 도형을 3D로 표시한다.

**입력**:
- 마커 감지 이벤트
- 레벨 ID

**처리**:
1. 레벨 데이터에서 도형 정보 로드
2. GeometryShapes.create() 호출
3. 도형 엔티티 생성
4. 마커에 도형 추가
5. 회전 애니메이션 시작

**출력**:
- 3D 도형 표시
- 도형 이름 라벨
- 도형 정보 패널

**성능 요구사항**:
- 마커 감지 후 3초 이내 도형 표시
- 30 FPS 이상 유지

#### FR-AR-003: 마커리스 AR
**우선순위**: 중간  
**설명**: 평면을 감지하여 마커 없이 도형을 배치한다.

**입력**:
- 카메라 뷰
- 평면 감지 데이터

**처리**:
1. 평면 감지 알고리즘 실행
2. 사용자 터치 위치 감지
3. 도형 배치
4. 도형 이동/회전/크기 조절

**출력**:
- 평면에 배치된 도형
- 조작 UI (이동, 회전, 크기)

### 4.2 레벨 시스템

#### FR-LVL-001: 레벨 선택
**우선순위**: 높음  
**설명**: 사용자는 7개 레벨 중 원하는 레벨을 자유롭게 선택할 수 있다.

**입력**:
- 사용자의 레벨 선택 (클릭/터치)

**처리**:
1. levels.json에서 레벨 데이터 로드
2. 레벨 정보 표시 (이름, 난이도, 설명)
3. 진행도 표시 (별점, 완료 여부)

**출력**:
- 레벨 선택 화면
- 레벨별 상세 정보
- 시작 버튼

**제약사항**:
- 모든 레벨 처음부터 잠금 해제 (자유 선택)

#### FR-LVL-002: 레벨 진행
**우선순위**: 높음  
**설명**: 선택한 레벨의 학습 콘텐츠를 순차적으로 제공한다.

**입력**:
- 레벨 ID
- 사용자 진행 상태

**처리**:
1. AR 학습 단계 시작
2. 도형 관찰 및 학습
3. 퀴즈 단계 진행
4. 결과 평가

**출력**:
- 단계별 안내
- 진행률 표시
- 완료 시 보상

### 4.3 퀴즈 시스템

#### FR-QZ-001: 퀴즈 로드
**우선순위**: 높음  
**설명**: 레벨에 해당하는 퀴즈 문제를 로드한다.

**입력**:
- 레벨 ID

**처리**:
1. quiz-questions.json에서 문제 로드
2. 문제 순서 섞기 (선택사항)
3. 첫 번째 문제 표시

**출력**:
- 문제 텍스트
- 4개 선택지
- 진행률 (예: 1/10)

#### FR-QZ-002: 정답 체크
**우선순위**: 높음  
**설명**: 사용자의 답변을 체크하고 피드백을 제공한다.

**입력**:
- 사용자 선택 (0-3)
- 정답 인덱스

**처리**:
1. 선택과 정답 비교
2. 정답 시:
   - 점수 증가
   - 코인 지급
   - 정답 애니메이션
3. 오답 시:
   - 오답 피드백
   - 힌트 제공 옵션

**출력**:
- 정답/오답 메시지
- 해설 표시
- 다음 문제 버튼

**성능 요구사항**:
- 즉시 피드백 (< 100ms)

#### FR-QZ-003: 점수 계산
**우선순위**: 중간  
**설명**: 퀴즈 완료 후 점수를 계산하고 별점을 부여한다.

**입력**:
- 정답 수
- 소요 시간
- 힌트 사용 횟수

**처리**:
1. 기본 점수 = (정답 수 / 전체 문제) × 100
2. 시간 보너스 계산
3. 힌트 페널티 적용
4. 별점 결정:
   - 3성: 90점 이상
   - 2성: 70-89점
   - 1성: 60-69점
   - 0성: 60점 미만

**출력**:
- 총점
- 별점
- 상세 통계

### 4.4 진행도 추적

#### FR-PRG-001: 데이터 저장
**우선순위**: 높음  
**설명**: 사용자의 학습 진행도를 LocalStorage에 저장한다.

**입력**:
- 레벨 완료 데이터
- 점수 및 별점
- 획득 아이템

**처리**:
1. 기존 데이터 로드
2. 새 데이터 병합
3. LocalStorage에 저장
4. 저장 성공 확인

**출력**:
- 저장 완료 메시지

**데이터 구조**:
```javascript
{
  userId: "uuid",
  levels: {
    1: { completed: true, stars: 3, score: 95, time: 180 },
    2: { completed: true, stars: 2, score: 75, time: 240 }
  },
  coins: 500,
  xp: 1200,
  level: 5,
  cards: ["triangle-card", "square-card"],
  badges: ["explorer", "genius"]
}
```

#### FR-PRG-002: 통계 표시
**우선순위**: 중간  
**설명**: 사용자의 학습 통계를 시각화하여 표시한다.

**입력**:
- 저장된 진행도 데이터

**처리**:
1. 데이터 분석
2. 통계 계산:
   - 총 플레이 시간
   - 레벨 완료율
   - 평균 정답률
   - 획득 코인/경험치

**출력**:
- 통계 대시보드
- 그래프 (선택사항)
- 성취도 표시

### 4.5 게이미피케이션

#### FR-GM-001: 코인 시스템
**우선순위**: 중간  
**설명**: 사용자는 활동을 통해 코인을 획득하고 사용할 수 있다.

**획득 방법**:
- 퀴즈 정답: 10코인
- 레벨 클리어: 50코인
- 3성 클리어: 100코인 보너스
- 일일 미션: 200코인

**사용 방법**:
- 힌트 카드: 50코인
- 시간 연장: 30코인
- 캐릭터 스킨: 200코인

#### FR-GM-002: 경험치 및 레벨업
**우선순위**: 중간  
**설명**: 경험치를 획득하여 레벨업한다.

**입력**:
- 활동 완료 이벤트

**처리**:
1. 경험치 획득
2. 현재 레벨 확인
3. 레벨업 조건 체크
4. 레벨업 시 보상 지급

**출력**:
- 경험치 바 업데이트
- 레벨업 애니메이션
- 보상 알림

**레벨업 공식**:
```
필요 경험치 = 100 × (현재 레벨)^1.5
```

#### FR-GM-003: 도형 카드 수집
**우선순위**: 중간

## 6. 데이터 요구사항

### 6.1 데이터 모델

#### 6.1.1 Level Data Model

```javascript
{
  id: Number,              // 레벨 ID (1-7)
  name: String,            // 레벨 이름
  difficulty: Number,      // 난이도 (1-5)
  icon: String,            // 아이콘 (이모지)
  description: String,     // 설명
  targetGrade: String,     // 대상 학년
  curriculum: Array,       // 교육과정 코드
  shapes: Array,           // 도형 목록
  quizCount: Number,       // 퀴즈 문제 수
  timeLimit: Number,       // 제한 시간 (초)
  requiredScore: Number,   // 통과 점수
  rewards: Object          // 보상 (코인, 경험치, 카드)
}
```

#### 6.1.2 Quiz Question Model

```javascript
{
  id: String,              // 문제 ID
  type: String,            // 문제 유형 (multiple-choice)
  question: String,        // 문제 텍스트
  shape: String,           // 관련 도형
  shapeColor: String,      // 도형 색상
  options: Array,          // 선택지 (4개)
  answer: Number,          // 정답 인덱스 (0-3)
  explanation: String,     // 해설
  hint: String,            // 힌트
  difficulty: Number,      // 난이도 (1-3)
  points: Number           // 배점
}
```

#### 6.1.3 User Progress Model

```javascript
{
  userId: String,          // 사용자 ID (UUID)
  createdAt: Timestamp,    // 생성 시간
  lastPlayed: Timestamp,   // 마지막 플레이
  levels: Object,          // 레벨별 진행도
  coins: Number,           // 보유 코인
  xp: Number,              // 경험치
  level: Number,           // 사용자 레벨
  cards: Array,            // 수집한 카드
  badges: Array,           // 획득한 배지
  stats: Object            // 통계 데이터
}
```
